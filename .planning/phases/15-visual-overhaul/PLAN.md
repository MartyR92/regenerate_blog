# Visual Engine Overhaul Implementation Plan

## Overview
The current visual pipeline generates unacceptable "AI slop" for technical diagrams (via hardcoded SVG fallback or poor prompt engineering) and generic, disconnected Unsplash stock photos for header images. This plan outlines the complete replacement of these mechanics. We will integrate NanoBanana API for high-end, text-free avantgarde header images. For diagrams, we will implement a dual-path system: interactive Echarts for data metrics and Python-rendered Matplotlib/Plotly SVGs for conceptual architectures, ensuring all visuals adhere to the "Organic Precision" brand guidelines.

## Scope Definition (CRITICAL)
### In Scope
- Rewrite `scripts/visual.py` to use NanoBanana API for header images (no text inside images, high-end abstract aesthetics).
- Create a Hugo shortcode (`layouts/shortcodes/echarts.html`) to render interactive Echarts JSON data.
- Rewrite `scripts/image.py` to:
  - Generate Echarts JSON for data-focused metrics.
  - Generate and execute Python scripts (Matplotlib/Plotly) for system/architectural SVGs.
- Update GitHub Actions workflows (`image-agent.yml`, `visual-agent.yml`) to include necessary Python dependencies (e.g., `matplotlib`, `plotly`).
- Remove Unsplash dependencies and old procedural SVG fallback logic.

### Out of Scope (DO NOT TOUCH)
- Changes to the Writer Agent or how articles are drafted.
- Changes to the Editor Agent's review logic.
- Migration to other Static Site Generators.
- Unrelated CSS or Hugo theme modifications outside the `echarts` shortcode.

## Current State Analysis
- `scripts/visual.py` (Lines ~50-80): Currently prompts Gemini for a 1-3 word search query and calls Unsplash API.
- `scripts/image.py` (Lines ~90-150): Prompts Gemini to write raw `<svg>` code. If it fails, falls back to a hardcoded `generate_procedural_svg()` method which draws a low-quality black box.
- `layouts/`: Missing any shortcodes for rendering JS-based data visualization libraries.
- `.github/workflows/image-agent.yml`: Missing data science/plotting libraries.

## Implementation Phases

### Phase 1: NanoBanana Header Integration
- **Goal**: Replace generic Unsplash photos with bespoke, text-free avantgarde art generated by NanoBanana.
- **Steps**:
  1. [ ] Modify `scripts/visual.py` to import `requests` and call the NanoBanana generation API.
  2. [ ] Update the Gemini prompt in `visual.py` to extract core concepts from the article and synthesize a highly descriptive, aesthetic-focused prompt for NanoBanana (strictly enforcing "NO text, avantgarde, organic precision, dark mode" constraints).
  3. [ ] Handle the NanoBanana API response, download the generated image, and save it as `cover.jpg` (or `.webp`).
  4. [ ] Remove all Unsplash API logic and secrets handling.
- **Verification**: Run `python scripts/visual.py` locally on a sample markdown file. Verify a high-quality, text-free image is generated and linked in the frontmatter.

### Phase 2: Hugo Echarts Shortcode & Asset Setup
- **Goal**: Enable the Hugo site to render interactive Echarts from JSON data blocks.
- **Steps**:
  1. [ ] Create `layouts/shortcodes/echarts.html`.
  2. [ ] Write the HTML/JS logic inside the shortcode to initialize an Apache Echarts instance, parse the `.Inner` content as JSON, and render the chart.
  3. [ ] Inject a predefined "Organic Precision" dark-mode theme into the Echarts initialization to ensure brand consistency.
- **Verification**: Manually add an `{{< echarts >}} {"title": {"text": "Test"}} {{< /echarts >}}` block to a draft. Serve Hugo locally and verify the chart renders correctly in dark mode.

### Phase 3: Technical Diagrams (Data vs. System Logic)
- **Goal**: Rewrite the Image Agent to intelligently route visual generation to either Echarts JSON or Python-rendered SVGs.
- **Steps**:
  1. [ ] Rewrite `scripts/image.py`. Introduce routing logic: If the visual task is a "Technical Chart" (data/metrics), instruct Gemini to output Echarts Option JSON.
  2. [ ] If the visual task is a "Technical Diagram" (architecture/flow), instruct Gemini to output a Python script using Matplotlib/Plotly with strict styling constraints.
  3. [ ] Implement a safe execution wrapper in Python to run the generated script, capture the resulting SVG, and save it to the static folder.
  4. [ ] Update markdown insertion logic: Echarts JSON gets wrapped in the new shortcode; SVGs use the standard markdown image syntax.
  5. [ ] Remove the old `generate_procedural_svg` function.
- **Verification**: Run `python scripts/image.py` against a draft. Verify it produces valid JSON in the markdown for charts, and physically generates a beautiful SVG file for diagrams.

### Phase 4: Pipeline Dependency Updates
- **Goal**: Ensure the GitHub Actions runners have the necessary libraries to execute the new Python plotting scripts.
- **Steps**:
  1. [ ] Update `.github/workflows/image-agent.yml` to `pip install matplotlib plotly pandas seaborn`.
  2. [ ] Ensure any system-level dependencies for rendering headless SVGs (like `libgl1-mesa-glx` if required by Matplotlib/Qt) are added to the `apt-get install` step.
- **Verification**: Trigger the `Image Agent` workflow on GitHub and verify it completes without dependency or import errors.
