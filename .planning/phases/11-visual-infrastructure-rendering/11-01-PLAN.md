<task type="auto">
  <name>Task 2: API Error Handling, Synchronous WebP Generation, and Markdown Pathing</name>
  <files>scripts/image.py</files>
  <action>
    Update `scripts/image.py` to implement explicit error handling for Interactions API calls: catch rate limits (429) or service unavailability (5xx) exceptions, log a clear error report, and immediately halt the pipeline using `sys.exit(1)`.
    Update the script to synchronously generate a WebP fallback using standard CLI tools: import `cairosvg`, `subprocess`, and `tempfile`. After receiving the SVG string, save the SVG. Use `cairosvg.svg2png(url=svg_path, write_to=temp_png_path)` to write a temporary PNG file. Then, use `subprocess.run(['cwebp', '-q', '80', temp_png_path, '-o', webp_path], check=True)` to convert the PNG to WebP format. Clean up the temporary PNG.
    If either SVG or WebP generation fails, halt immediately using `sys.exit(1)`.
    Update the markdown injection logic to strictly use the root-relative path: `![{caption}](/blog/images/{year}/{slug}/{img_filename})`.
    Update the prompt sent to Gemini: instruct it to use embedded CSS (`currentColor` or `@media (prefers-color-scheme: dark)`) for dark/light mode compatibility, use `@media (max-width: 600px)` for mobile text resizing, enforce a maximum width constraint, and keep a scalable `viewBox`.
  </action>
  <verify>grep "cwebp" scripts/image.py && grep "prefers-color-scheme" scripts/image.py && grep "sys.exit(1)" scripts/image.py</verify>
  <done>The script handles Interactions API errors, reliably outputs `.svg` and `.webp` using standard CLI tools, handles dark/light mode, and inserts the correct `/blog/` prefixed path.</done>
</task>
