# Research: Phase 8 - Diagram Rendering & Language Fixes

## 1. Rendering Issue Analysis
- **Observed behavior:** Technical diagrams are not displaying in the blog posts.
- **Root Cause:** The `scripts/image.py` saves images to `assets/images/`. In Hugo, files in `assets/` are part of the asset pipeline and must be accessed via `.Resources.Get` or Hugo shortcodes. Standard Markdown syntax `![alt](/images/...)` looks for files in the `static/` directory (which maps to the web root `/`).
- **Solution:** Change the output directory in `scripts/image.py` to `static/images/`. This ensures that `/images/YYYY/slug/file.svg` is accessible to the browser.

## 2. Linguistic Inconsistency
- **Observed behavior:** Post is in German, but the diagram caption/alt-text is in English.
- **Root Cause:** The caption string in `scripts/image.py` is hardcoded in English: `*Figure 1: Technical visualization generated by image-agent.*`.
- **Solution:** 
    - The script should parse the draft's Front Matter or content to detect the language.
    - It should then request a localized caption from Gemini or use a localized template.
    - Remove "generated by image-agent" to maintain a professional, reader-facing tone.

## 3. Reader-Facing Captions
- **Requirement:** Captions must describe what is *on* the diagram (e.g., "Visualization of carbon sequestration rates in various soil types").
- **Implementation:** Update the prompt to Gemini 1.5 Flash to include: "Provide a concise, reader-facing caption for this diagram in [LANGUAGE] that describes the visual data."

## 4. Path Logic Verification
- Ensure that the `year` and `slug` variables are correctly derived from the file path/metadata to prevent broken links in the future.
