# Frontend Overhaul: Hugo to Astro Migration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Migrate the existing Hugo-based `blowfish` frontend to a custom Astro + Tailwind + React architecture that strictly implements the dual-identity (Macro/Micro) brand guidelines, including "Leaf Laminate" styling and "The Void" aesthetics.

**Architecture:** We are replacing Hugo with Astro. We will use Astro's `Content Collections` to parse the existing Markdown files (`content/de/posts` and `content/en/posts`) generated by the PydanticAI orchestrator. We will build a global React context/state to handle the "Macro/Micro" Level Toggle, mapping Tailwind CSS utility classes to the brand guidelines (e.g., `--void-primary`, `EB Garamond`). 

**Tech Stack:** Astro 5.x, React 18, Tailwind CSS v4, Lenis (Smooth Scroll), Framer Motion (Transitions).

### Task 3: Setup Astro Content Collections & Routing

**Files:**
- Create: `src/content/config.ts`
- Modify: `src/pages/index.astro`
- Create: `src/pages/[lang]/posts/[slug].astro`

**Step 1: Define Content Schema**
Write into `src/content/config.ts`:
```typescript
import { defineCollection, z } from 'astro:content';

const blogCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    language: z.enum(['de', 'en']),
    date: z.date().or(z.string()),
    draft: z.boolean().default(false),
    description: z.string().optional(),
    tags: z.array(z.string()).optional(),
    featureImage: z.string().optional(),
  }),
});

export const collections = {
  'de': blogCollection,
  'en': blogCollection,
};
```

**Step 2: Migrate Markdown Files into Collections**
Run: `mkdir -p src/content/de src/content/en`
Run: `mv content/de/posts/* src/content/de/`
Run: `mv content/en/posts/* src/content/en/`
Run: `rm -rf content/`

**Step 3: Create Dynamic Route for Posts**
Write into `src/pages/[lang]/posts/[slug].astro`:
```astro
---
import { getCollection } from 'astro:content';
import Layout from '../../../layouts/Layout.astro';

export async function getStaticPaths() {
  const dePosts = await getCollection('de');
  const enPosts = await getCollection('en');
  
  const mapPaths = (posts, lang) => posts.map(post => ({
    params: { lang, slug: post.slug },
    props: { post },
  }));

  return [...mapPaths(dePosts, 'de'), ...mapPaths(enPosts, 'en')];
}

const { post } = Astro.props;
const { Content } = await post.render();
---

<Layout title={post.data.title}>
  <article class="max-w-4xl mx-auto py-20 px-4">
    <h1 class="font-garamond text-5xl md:text-7xl text-brass-primary mb-8">{post.data.title}</h1>
    <div class="prose prose-invert prose-brass max-w-none">
      <Content />
    </div>
  </article>
</Layout>
```

### Task 4: Build Layout & React Theme Toggle (Macro/Micro)

**Files:**
- Create: `src/layouts/Layout.astro`
- Create: `src/components/ThemeToggle.tsx`

**Step 1: Write Theme Toggle React Component**
Write into `src/components/ThemeToggle.tsx`:
```tsx
import React, { useState, useEffect } from 'react';

export default function ThemeToggle() {
  const [theme, setTheme] = useState<'macro' | 'micro'>('micro');

  useEffect(() => {
    document.body.className = `theme-${theme}`;
  }, [theme]);

  return (
    <button 
      onClick={() => setTheme(theme === 'macro' ? 'micro' : 'macro')}
      className="fixed bottom-8 right-8 z-50 px-6 py-3 rounded-sm leaf-laminate brass-hover font-syne text-sm uppercase tracking-widest"
    >
      {theme === 'macro' ? 'Rə:NatureFørce' : 'Rə:Vive La Palma'}
    </button>
  );
}
```
